# Mais sobre mim

## Minha jornada até aqui

Entrei na área de tecnologia pelo **curso técnico em Azure no SENAI**. Antes disso, não tinha contato com programação. O curso abriu uma porta que eu nem sabia que existia.

A partir daí, fui guiado por um **mentor desenvolvedor sênior** que montou uma [trilha de estudos completa (disponível em marcelao.dev)](https://marcelao.dev). Ele me ensinou não só tecnologia, mas o que realmente importa: como pensar como um desenvolvedor, como resolver problemas, como se comunicar, e como entregar valor.

Foram 3 anos de estudo e prática contínua. Cada projeto foi um degrau: a API fake com CRUDCRUD me ensinou fundamentos de HTTP e JavaScript, o freela PHP me mostrou como é trabalhar com código em produção, e o Pescador IA foi onde tudo se consolidou — arquitetura, refatoração, boas práticas, trabalho com Docker, PostgreSQL, Laravel avançado.

## Como trabalho

### Planejamento antes de execução
Gosto de entender o problema antes de escrever uma linha de código. Qual é a real necessidade? Quem vai usar? Qual o fluxo? Com isso claro, desenho as etapas e começo a construir.

### Comunicação é prioridade
Não adianta código bonito se ninguém entende o que foi feito. Gosto de manter comunicação aberta com a equipe, alinhar expectativas, e documentar o que é relevante.

### Adaptação rápida
Cada projeto tem seu contexto. Já trabalhei com código legado cheio de dívida técnica, com sistemas novos que precisavam sair do papel, e com integrações que exigiam aprender algo novo rapidamente. Me adapto ao que o projeto precisa.

### Colaboração
Aprendi muito com mentorias e trocas com outros desenvolvedores. Levo isso pro trabalho: colaborar, revisar código, dar e receber feedback faz parte do processo.

### Cuidado com o legado
Não basta entregar funcionalidade nova — o código precisa ser sustentável. Refatorar, pagar dívida técnica, deixar o ambiente melhor do que encontrei. É assim que projetos escalam.
